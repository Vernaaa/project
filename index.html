Email 提醒邏輯建議

這裡說明 GitHub Pages 版本與正式寄信版本的差異。
目前 GitHub Pages 版本可做到
1. 管理收件人與提醒規則
2. 將設定儲存在瀏覽器 localStorage
3. 匯出 JSON 供後續串接後端使用
若要真的自動寄信，建議追加
1. 後端 API（Node.js / Apps Script / Supabase Edge Function）
2. 排程服務（cron）
3. 郵件服務（Resend / SendGrid / Gmail SMTP）

也就是說，GitHub Pages 可以當前台入口；真正寄信要靠後端，不然瀏覽器自己不會長出郵差。
